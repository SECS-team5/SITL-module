# Анализ безопасности системы SITL-module (Дрон-контроллер)

## Пункт 1. Ключевые активы, оценка уровня ущерба и приемлемость риска

### 1.1. Идентификация активов

| №  | Актив                    | Описание                                    | Категория    |
|----|--------------------------|---------------------------------------------|--------------|
| A1 | Redis `SITL:DRONE:*`     | Все активные дроны (pattern scan, TTL 2ч)   | Хранилище    |
| A2 | Kafka `verified-messages`| Команды с `verifier_stage: "SITL-v1"`       | Данные       |
| A3 | Дрон-контроллер сервис   | 10 Гц обновление позиций + команды          | Сервис       |
| A4 | MQTT `sitl/commands`     | NMEA от ГНСС-приемника                      | Данные       |
| A5 | Координаты СПб-42/WGS84  | Конвертация систем координат (x,y,z)        | Вычисления   |
| A6 | Docker `.env` секреты    | `REDIS_PASS`, Kafka SASL                    | Секреты      |
| A7 | `verifier.py` сервис     | Валидация `SITL-v1` метки                   | Сервис       |

### 1.2. Оценка уровня ущерба

| Актив              | Конфиденц.  | Целостность | Доступность | Обоснование                    |
|--------------------|-------------|-------------|-------------|--------------------------------|
| **A1 Redis DRONE:**| **Критический** | **Критический** | **Критический** | SCAN всех дронов = флот     |
| **A2 Kafka SITL-v1**| Средний     | **Критический** | Высокий    | Фейк верификатор = RCE        |
| **A3 Контроллер 10Гц**| Средний  | **Критический** | **Критический** | 1 сбой = потеря контроля |
| A4-A7              | Средний     | Высокий     | Высокий     | GNSS spoofing                 |

### 1.3. Приемлемость риска

| Актив         | Приемлемость риска | Пояснение                     |
|---------------|--------------------|-------------------------------|
| **A1 Redis SCAN** | **Неприемлем**   | 1000 SCAN/sec = Redis OOM     |
| **A3 10 Гц**  | **Неприемлем**    | Deadline miss = флот offline  |
| **A2 SITL-v1**| **Неприемлем**    | Replay attack на команды      |

## Пункт 2. Роли пользователей и ключевые сценарии

### 2.1. Роли пользователей

| Роль              | Описание                     | Аутентификация     | Права                          |
|-------------------|------------------------------|--------------------|--------------------------------|
| Верификатор       | Отправляет `SITL-v1` команды | Kafka SASL         | Только `verified-messages`     |
| Дрон-контроллер   | 10 Гц Redis + Kafka          | Docker internal    | Чтение `SITL:DRONE:*` + Kafka  |
| ГНСС-приемник     | NMEA данные                  | MQTT SASL          | Только `sitl/commands`         |
| Разработчик       | make up/down/test            | Docker socket      | Полный доступ                  |
