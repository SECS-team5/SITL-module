# 1. Центры Принятия Безопасных решений (ЦПБ)

## Обзорная схема ЦПБ

**ЦПБ-1: ЦЕНТР ВЕРИФИКАЦИИ КОМАНД**

Компонент: sitl_verifier
Входные топики: sitl.commands, sitl-drone-home
Выходные топики: sitl.verified-commands, sitl.verified-home

Решения:
- Пропустить или отклонить входное сообщение
- Классификация по типу топика
- Валидация по JSON Schema

ДВБ: Высокая
Описание: Защитный барьер на входе системы, определяет качество всех последующих решений

Поток: verified messages → ЦПБ-2 / ЦПБ-3

---

**ЦПБ-2: ЦЕНТР ДОВЕРЕННОЙ ДОМАШНЕЙ ПОЗИЦИИ**

Компонент: sitl_controller (при обработке HOME)
Вход: sitl.verified-home
Выход: Redis drone:{id}:state (home set)

Решения:
- Установить home-позицию дрона
- Инициализировать состояние ARMED
- Определить геозону безопасности

ДВБ: Средняя
Описание: Критическая точка для COMMAND authorization

Поток: state in Redis → ЦПБ-3

---

**ЦПБ-3: ЦЕНТР ПОЛИТИК МАРШРУТИЗАЦИИ**

Компонент: sitl_controller (при обработке COMMAND)
Вход: sitl.verified-commands + существующее состояние
Выход: Обновлённое Redis drone:{id}:state

Решения:
- Применить команду к состоянию
- Определить статус: ARMED vs MOVING
- Задать вектор движения (vx, vy, vz)

ДВБ: Высокая
Описание: Непосредственно влияет на физическое поведение дронов

Поток: → ЦПБ-4, ЦПБ-5, ЦПБ-6

---

**ЦПБ-4: ЦЕНТР БЕЗОПАСНОЙ ТЕЛЕМЕТРИИ**

Компонент: sitl_messaging

Решения:
- Возврат координат
- Отказ при отсутствии состояния

ДВБ: Высокая

---

**ЦПБ-5: ЦЕНТР ИЗОЛЯЦИИ СООБЩЕНИЙ**

Компонент: sitl_messaging

Решения:
- Пропуск запроса позиции
- Валидация drone_id
- Формирование ответа

ДВБ: Средняя

---

**ЦПБ-6: ЦЕНТР ОБНОВЛЕНИЯ ПОЗИЦИЙ**

Компонент: sitl_core
Вход: Redis drone:*:state (MOVING)
Выход: Обновлённые координаты в Redis

Решения:
- Обновлять или нет (проверка статуса)
- Расчёт новых координат по вектору
- Продление TTL

ДВБ: Средняя
Описание: Операционный компонент, не влияет на decisions напрямую

---

## Детальное описание ЦПБ

### ЦПБ-1: Центр верификации команд

**Компонент:** sitl_verifier

| Параметр | Значение |
|----------|----------|
| Входные топики | sitl.commands, sitl-drone-home |
| Выходные топики | sitl.verified-commands, sitl.verified-home |
| Критичность | Высокая |
| Атакуемая поверхность | Все входные сообщения проходят через этот центр |

**Функции принятия решений:**

1. Парсинг JSON — отклонение при невалидном JSON
2. Классификация по топику — определение типа сообщения
3. Валидация по схеме — проверка структуры сообщения
4. Публикация верифицированного сообщения

**Политики безопасности:**
- Только JSON Schema валидация
- Reject неструктурированных сообщений
- Логирование всех отклонённых сообщений

---

### ЦПБ-2: Центр доверенной домашней позиции

**Компонент:** sitl_controller (при обработке HOME)

| Параметр | Значение |
|----------|----------|
| Вход | sitl.verified-home (верифицированное сообщение) |
| Выход | Redis drone:{id}:state с заполненными home_* полями |
| Критичность | Средняя |
| Зависимость от других ЦПБ | Не зависит (инициатор состояния) |

**Функции принятия решений:**

1. Повторная валидация (belt-and-suspenders)
2. Построение состояния из верифицированного HOME
3. Сохранение в Redis с установкой home-позиции

**Политика безопасности:** Только после HOME возможны COMMAND

---

### ЦПБ-3: Центр политик маршрутизации

**Компонент:** sitl_controller (при обработке COMMAND)

| Параметр | Значение |
|----------|----------|
| Вход | sitl.verified-commands + существующее состояние из Redis |
| Выход | Обновлённое Redis drone:{id}:state |
| Критичность | Высокая |
| Зависимость от других ЦПБ | Требует ЦПБ-2 (HOME must exist) |

**Политика безопасности:**
- COMMAND без HOME = IGNORE
- Проверка диапазонов скоростей (vx/vy: ±50, vz: ±10)
- Статус ARMED требуется для приёма команд

---

### ЦПБ-4: Центр безопасной телеметрии

**Компонент:** sitl_messaging

| Параметр | Значение |
|----------|----------|
| Вход | sitl.telemetry.request |
| Выход | sitl.telemetry.response или reply_to |
| Критичность | Высокая |
| Операция | Read-only из Redis |

**Функции принятия решений:**

1. Валидация запроса по схеме
2. Поиск состояния дрона в Redis
3. Формирование ответа с координатами
4. Публикация в reply_to или default топик

**Политика безопасности:**
- Только READ операции
- Ответ только в указанный reply_to
- Сохранение correlation_id для traceability

---

### ЦПБ-5: Центр изоляции сообщений

**Компонент:** Системная шина (SystemBus)

| Параметр | Значение |
|----------|----------|
| Вход | Все входящие сообщения |
| Выход | Маршрутизация по топикам |
| Критичность | Средняя |
| Ответственность | Разделение потоков команд/телеметрии/событий |

**Политика изоляции:**

| Топик | Потребители | Тип данных |
|-------|-------------|------------|
| sitl.commands | verifier | Raw commands |
| sitl-drone-home | verifier | Raw home |
| sitl.verified-commands | controller | Trusted commands |
| sitl.verified-home | controller | Trusted home |
| sitl.telemetry.request | messaging | Position requests |
| sitl.telemetry.response | External systems | Position responses |

---

### ЦПБ-6: Центр обновления позиций

**Компонент:** sitl_core

| Параметр | Значение |
|----------|----------|
| Вход | Redis state (только MOVING дроны) |
| Выход | Обновлённые lat/lon/alt |
| Частота | 10 Hz |
| Критичность | Средняя |

**Политика безопасности:**
- SCAN с COUNT для предотвращения блокировки Redis
- Обновляет только MOVING дроны
- TTL продлевается при каждом обновлении

## Сводная таблица ЦПБ

| ЦПБ | Название | Компонент | ДВБ | Размер кода | Сложность |
|-----|----------|-----------|-----|-------------|-----------|
| 1 | Центр верификации | sitl_verifier | Высокая | ~170 LOC | Средняя |
| 2 | Центр домашней позиции | controller (HOME) | Средняя | ~40 LOC | Низкая |
| 3 | Центр маршрутизации | controller (CMD) | Высокая | ~60 LOC | Средняя |
| 4 | Центр телеметрии | sitl_messaging | Высокая | ~140 LOC | Средняя |
| 5 | Центр изоляции | SystemBus | Средняя | ~200 LOC | Высокая |
| 6 | Центр обновления | sitl_core | Средняя | ~110 LOC | Низкая |

**Обозначения:**
- ДВБ — Доверие к безопасности (Trustworthiness)
- Размер кода — приблизительное количество строк в логике принятия решений
- Сложность — вычислительная и логическая сложность политик
